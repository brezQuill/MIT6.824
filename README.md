## Lab2: Raft
Lab2将实现 Raft——一种复制状态机协议

复制服务通过在多台副本服务器上存储其状态（即数据）的完整副本来实现容错。即使部分服务器发生故障（崩溃、网络中断或不稳定），副本也能让服务持续运行。其挑战在于，故障可能导致各副本持有的数据不一致。

Raft 将客户端请求组织成一个序列（称为日志），并确保所有副本服务器看到相同的日志。每个副本按日志顺序执行客户端请求，并将其应用到服务的本地状态副本中。由于所有存活的副本都看到相同的日志内容，它们会以相同的顺序执行相同的请求，从而保持一致的服务状态。如果一台服务器发生故障后恢复，Raft 会负责将其日志更新到最新状态。只要多数服务器存活并能相互通信，Raft 就能持续运行。如果无法形成多数派，Raft 将无法正常工作，但一旦多数派恢复通信，它便能从断点继续。

在本实验中，将把 Raft 实现为一个 Go 对象类型及关联方法，旨在作为更大服务中的一个模块。一组 Raft 实例通过 RPC​ 相互通信，以维护复制的日志。Raft 的接口将支持一个带编号的命令（也称为日志条目）组成的序列。序列中的条目按索引号编号。这些日志条目最终会被提交， Raft 应将已提交的日志条目发送给上层服务执行。
